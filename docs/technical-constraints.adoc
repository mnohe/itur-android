== Authentication

[cols="1,4"]
|===
| Type | Details

| Anonymous
| Device UUID stored in `EncryptedSharedPreferences` (AES256-GCM master key, prefix `itur_secure_prefs`).
  All users start as anonymous. UUID is generated once and persists across sessions.

| Registered (Google)
| AndroidX Credentials API â†’ `GoogleIdTokenCredential` â†’ Firebase Auth.
  Required to *start* activities. Optional for joining.
  `GOOGLE_WEB_CLIENT_ID` must be present in `local.properties`.
|===

== Required permissions

[cols="2,2,3"]
|===
| Permission | When Requested | Required For

| `ACCESS_FINE_LOCATION`
| On `MapScreen` entry (via `LaunchedEffect`)
| Location tracking, map centering, writing to Firestore

| `CAMERA`
| When "Join an activity" is tapped (via `LaunchedEffect` inside `QRScanSheet`)
| QR barcode scanning via ML Kit

| `INTERNET`
| Declared in manifest; granted at install
| Firebase Firestore reads/writes, MapLibre tile downloads
|===

NOTE: Camera hardware is declared `required="false"` in the manifest.
If the camera permission is denied, a toast informs the user that QR scanning requires camera access and the scan sheet is not opened.
Tapping *Join an activity* again repeats the toast until the permission is granted.

[cols="2,2,3"]
|===
| Service | Role | Test Dependency

| Firebase Firestore
| Activity, location, and user data storage (real-time snapshot reads)

| Required on test device; Emulator must have Google Play

|===

== Known asynchronous and background operations

[cols="2,2,3"]
|===
| Operation | Mechanism | Notes for Testing

| Current user fetch on startup
| `viewModelScope.launch` in `MapViewModel.init`
| Race condition possible before first composition; use `ComposeTestRule.waitForIdle()`

| Location polling
| `FusedLocationProviderClient` callback on `Looper.getMainLooper()`, 2-second interval
| Stopped on `leaveActivity()`; a fake client is needed for deterministic tests (TODO MD13)

| Location write to Firestore
| `CoroutineScope(Dispatchers.IO).launch` (not `viewModelScope`)
| Lifecycle leak possible; not cancellable via ViewModel scope (TODO HI07)

| Firestore reads
| Suspend functions; snapshot reads (no real-time listener)
| Participant list is not auto-updated; requires manual refresh or re-query

| Google Sign-In
| Credential Manager coroutine; result on main thread
| Requires an `Activity` context; cannot be triggered from a `ViewModel` in isolation

| QR decoding
| ML Kit `ImageAnalysis` on a background executor (CameraX)
| Result posted back to main thread via `onScanSuccess` callback

| Attention request
| `viewModelScope.launch` â†’ Firestore write
| Silent; no UI feedback on success or failure
|===
